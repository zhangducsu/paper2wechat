# PDF论文 → 微信公众号文章 Agent工作组

## 一、项目概述

本项目是一个**PDF学术论文 → 微信公众号科普文章**的全自动化工作流，基于 Claude Code 多 Agent 架构实现，融合了 wechat-writing skill 的写作技巧和质检体系。

### 工作流程

```
PDF论文 → Agent 1(extractor) → 动态定调 → Agent 2(writer) → 选主题 → 渲染(md2wechat.py) → Agent 3(reviewer) → 最终HTML
         提取内容+图片        确认风格类型    撰写Markdown    选排版主题    Markdown→HTML       审核排版+商业闭环
```

### 命令体系

| 命令 | 用法 | 说明 |
|------|------|------|
| `/paper2wechat` | `/paper2wechat paper.pdf` | 完整工作流：提取→定调→写MD→选主题→渲染→审核 |
| `/extract-style` | `/extract-style 推文链接或HTML` | 从推文提取样式模板 + 拆解写作技巧，沉淀到知识库 |
| `/analyze-article` | `/analyze-article 推文链接或HTML` | 纯写作技巧拆解（不提取样式），沉淀到知识库 |

如果需要直接给微信推文链接让 Agent 自动获取内容（用于 `/extract-style` 和 `/analyze-article` 命令），请部署 wechat-parser-mcp。

#### 4.1 安装 wechat-parser-mcp

```bash
# 1. 克隆项目
git clone https://github.com/Xiaojun928/wechat-parser-mcp.git
cd wechat-parser-mcp

# 2. 安装（含浏览器支持，推荐）
pip install -e ".[browser]"
playwright install chromium
```

#### 4.2 配置 Claude Code MCP

编辑 Claude Code 的 MCP 配置文件，添加 wechat-parser：

**配置文件位置**：
- Windows: `%APPDATA%\Claude\claude_desktop_config.json`
- macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
- Linux: `~/.config/Claude/claude_desktop_config.json`

添加以下配置：

```json
{
  "mcpServers": {
    "wechat-parser": {
      "command": "python",
      "args": ["-m", "wechat_parser_mcp.server"],
      "env": {}
    }
  }
}
```

如果使用虚拟环境，将 `command` 改为虚拟环境的 Python 路径：

```json
{
  "mcpServers": {
    "wechat-parser": {
      "command": "/path/to/wechat-parser-mcp/venv/bin/python",
      "args": ["-m", "wechat_parser_mcp.server"],
      "env": {}
    }
  }
}
```

配置完成后**重启 Claude Code**。

#### 4.3 验证

在 Claude Code 中测试：

```
请帮我解析这篇微信公众号文章：https://mp.weixin.qq.com/s/xxxxx
```

如果成功返回文章标题、正文、图片等信息，说明部署成功。

> **备用方案**：如果未部署 MCP，可使用浏览器书签手动提取推文 HTML（见第八章）。

#### 4.4 常见故障排查

**① 抓取内容被截断（仅 ~2000 字符）**

根因：`parse_wechat_article` 工具的 `detail` 参数默认为 `concise`，会对正文做截断处理。

解决：调用时显式传入 `"detail": "detailed"`：
```json
{"url": "链接", "format": "json", "detail": "detailed", "extract_method": "auto"}
```
> 本项目 `/analyze-article` 和 `/extract-style` 命令已内置此参数。

**② Windows 终端输出中文乱码 / GBK 编码错误**

根因：Windows 终端默认使用 GBK 编码，MCP 返回的 Unicode 字符（如上标数字 `⁶`）无法编码。

解决：
- 方案A：启动 Python 时加 `-X utf8` 参数
- 方案B：将输出写入文件（`encoding='utf-8'`）而非打印到终端
- 方案C：在 Claude Code 中通过 MCP 协议调用（自动处理编码，不会出现此问题）

**③ MCP 工具在 Claude Code 中不可见**

检查清单：
1. MCP 配置文件位置是否正确：
   - Claude Code CLI：`~/.claude/mcp.json`（用户级）或 `.claude/mcp.json`（项目级）
   - Claude Desktop：`%APPDATA%\Claude\claude_desktop_config.json`
2. `command` 路径是否指向正确的 Python 解释器（Windows 需用正斜杠或双反斜杠）
3. 配置后是否**重启了 Claude Code**
4. 运行 `python -m wechat_parser_mcp.server` 是否能正常启动（无 import 错误）

**④ 直接调用 Python 模块时提示 `coroutine was never awaited`**

根因：`parse_article` 是 async 函数，不能同步调用。

解决：
```python
import asyncio
from wechat_parser_mcp.server import parse_article

async def main():
    result = await parse_article(url, format='json', detail='detailed')
    print(result)

asyncio.run(main())
```

## 五、使用方法

### 5.1 写文章：`/paper2wechat`

```
/paper2wechat path/to/paper.pdf
```

**完整执行流程**（8步）：

| 步骤 | 执行者 | 说明 |
|------|--------|------|
| 1 | 系统 | 创建临时工作目录 |
| 2 | Agent 1 (extractor) | 提取 PDF 内容（文本 + 图片 + 元信息） |
| 3 | 系统 | **向用户确认风格类型**（叙事转化风 / 标准八股风） |
| 4 | Agent 2 (writer) | 撰写 Markdown 文章 + 追加企业引流模块 |
| 5 | 系统 | **向用户确认排版主题**（doocs经典/优雅/简洁 / 自定义模板） |
| 6 | md2wechat.py | Markdown → HTML 渲染（注入内联样式） |
| 7 | Agent 3 (reviewer) | 审核排版（微信兼容性 + 商业闭环检查） |
| 8 | 系统 | 输出最终文件到 `output/` 目录 |

**发布到微信**：
1. 浏览器打开生成的 HTML 文件
2. `Ctrl+A` 全选 → `Ctrl+C` 复制
3. 微信公众号编辑器 → `Ctrl+V` 粘贴
4. 上传图片（替换本地图片路径）→ 预览 → 发布

### 5.2 提取样式 + 拆解技巧：`/extract-style`

```
/extract-style https://mp.weixin.qq.com/s/xxxxx
```

或直接粘贴推文 HTML 内容。

**执行流程**（6步）：

| 步骤 | 说明 |
|------|------|
| 1 | 获取推文内容（MCP自动抓取 → 浏览器抓取 → 提示备用方案） |
| 2 | 提取排版样式 → 保存为 `custom_YYYYMMDD.css` + `custom_YYYYMMDD_inline.json` |
| 3 | 三维拆解（结构模式 / 写作技巧 / 商业转化埋点） |
| 4 | 归类建档 → 输出拆解报告请用户确认 |
| 5 | 持久化沉淀到知识库（只追加不覆盖） |
| 6 | 报告结果 + 使用说明 |

### 5.3 纯拆解技巧：`/analyze-article`

```
/analyze-article https://mp.weixin.qq.com/s/xxxxx
```

与 `/extract-style` 相同的三维拆解流程，但**不提取样式模板**，只做写作技巧拆解和知识库沉淀。

## 六、核心功能详解

### 6.1 两种文章风格类型

| 类型 | 名称 | 目标读者 | 核心特点 |
|------|------|----------|----------|
| 类型一 | 叙事转化风 | 临床医生、非生信专业研究者 | 痛点共鸣、降维打击、商业转化 |
| 类型二 | 标准八股风 | 有生信外包需求的科研同行 | 严谨专业、模块化、技术推介 |

**类型一文章结构**：引子与痛点 → 核心突破拆解 → 落地场景 → 行业意义 → 商业承接

**类型二文章结构**：引言导读 → 文章档案库 → 核心小结 → 研究方法 → 结果图形 → 结论启示 → 技术推介

### 6.2 排版主题系统

| 主题 | 文件 | 风格 | 来源 |
|------|------|------|------|
| doocs/default | `doocs_default.css` | 经典风格 | doocs/md 开源项目 |
| doocs/grace | `doocs_grace.css` | 优雅风格 | doocs/md 开源项目 |
| doocs/simple | `doocs_simple.css` | 简洁风格 | doocs/md 开源项目 |
| custom_yijiyin | `custom_yijiyin_20250510.css` + `.json` | 蓝色系 (#0c6ffe) | 从真实推文提取 |

**主题工作原理**：
1. `md2wechat.py` 先加载 `doocs_base.css` 作为基础层，再加载具体主题作为覆盖层
2. 解析 CSS 规则，将 `var(--md-primary-color)` 等变量替换为实际值
3. 将 CSS 规则映射为 HTML 元素的内联 `style` 属性
4. `--primary-color` 参数同时控制主题色和加粗文字/引用块颜色

**自定义主题**：使用 `/extract-style` 命令从你喜欢的推文中提取样式，自动生成 CSS 模板和 JSON 映射文件。

### 6.3 知识库进化机制

知识库位于 `.claude/templates/references/` 目录，包含 4 个文件：

| 文件 | 内容 |
|------|------|
| `structure-patterns.md` | 文章结构模板（按类型分类） |
| `writing-techniques.md` | 写作技巧库（通用 + 各类型专属） |
| `article-checklist.md` | 质检清单（排版 + 各类型专属） |
| `company-footer.md` | 企业引流模板 |

**进化规则**：每次使用 `/extract-style` 或 `/analyze-article` 拆解推文后，新模式/新技巧以**"只追加不覆盖"**的方式沉淀到知识库中，实现持续进化。

### 6.4 商业闭环质检

Agent 3 (reviewer) 会检查以下商业转化要素：

- **引流矩阵**：文章中前部是否插入了同技术方向的"相关阅读"超链接
- **顺滑过渡**：结尾的企业服务介绍是否与前文解读的核心技术完美映射
- **转化组件**：联系电话、邮箱、企业微信、官网等基础设施是否展示完整

### 6.5 微信编辑器兼容性保障

最终 HTML 严格遵守以下规则，确保可直接粘贴到微信编辑器：

- 所有样式使用内联 `style` 属性（无外部 CSS、无 `<style>` 标签、无 class 选择器）
- 不使用 `position: fixed/sticky/absolute/relative`
- 不使用 `z-index`
- 不使用 JavaScript
- 不使用 `<video>` / `<audio>` / `<iframe>`
- 图片使用 `<img>` 标签（非 background-image）
- 字体使用系统字体栈

## 七、Agent 架构

```
┌──────────────────────────────────────────────────────────────┐
│                  主编排命令 (paper2wechat.md)                 │
│  .claude/commands/paper2wechat.md                           │
└────────┬──────────────────┬──────────────────┬───────────────┘
         │                  │                  │
   ┌─────▼──────┐    ┌──────▼──────┐   ┌──────▼──────┐
   │  Agent 1   │    │  Agent 2    │   │  Agent 3    │
   │ extractor  │───>│  writer     │──>│  reviewer   │
   │            │    │             │   │             │
   │ PDF内容提取 │    │ Markdown撰写│   │ 排版审核     │
   └────────────┘    └─────────────┘   └─────────────┘
         │                  │                  │
         ▼                  ▼                  ▼
   content.json       article.md          final.html
   figures/           (Markdown)          (可直接粘贴)
```

### Agent 1: extractor（PDF内容提取器）

- **输入**：PDF 文件路径
- **输出**：`.claude/tmp/extracted/` 下的结构化 JSON + 图片
- **工具**：`extract_figures.py`（PyMuPDF）、pdfplumber（文本提取）
- **提取内容**：论文元信息、章节文本、Figures（>200x200px）、Figure Legends、关键数据

### Agent 2: writer（科普文章撰写器）

- **输入**：extractor 输出的结构化内容 + 用户选择的风格类型
- **输出**：Markdown 文件（`.claude/tmp/extracted/article.md`）
- **参考**：`writing-techniques.md`（写作技巧）、`company-footer.md`（企业引流）
- **格式**：标准 Markdown，图片引用 `![描述](figures/fig1.png)`，图注 `**Fig. 1** 说明`

### Agent 3: reviewer（排版审核器）

- **输入**：渲染后的 HTML 文件
- **输出**：审核通过的最终 HTML + 审核报告
- **参考**：`article-checklist.md`（质检清单）
- **检查项**：图片完整性、图文引用一致性、微信兼容性、排版质量、内容质量、商业闭环

## 八、微信推文获取方案

### 方案一：wechat-parser-mcp（推荐）

配置后直接给链接即可自动获取，内置三层降级策略：

```
HTTP直接请求（快速） → 失败（反爬） → Playwright渲染（稳定） → 失败 → 提示备用方案
```

详见第四章步骤4。

### 方案二：浏览器书签（备用）

适用于未部署 MCP 或 MCP 抓取失败的情况。

1. 在浏览器中新建书签（`Ctrl+D`）
2. 编辑书签，将"网址"替换为以下代码（整行复制）：

```
javascript:void(function(){var c=document.getElementById('js_content');if(!c){alert('未找到正文区域');return}var h=c.outerHTML;navigator.clipboard.writeText(h).then(function(){alert('已复制！回到Claude Code粘贴即可。')}).catch(function(){var ta=document.createElement('textarea');ta.value=h;document.body.appendChild(ta);ta.select();document.execCommand('copy');document.body.removeChild(ta);alert('已复制！回到Claude Code粘贴即可。')})})();
```

3. 打开目标推文 → 点击书签 → 回到 Claude Code `Ctrl+V` 粘贴

### 方案三：手动复制源代码（最后手段）

1. 在推文页面按 `Ctrl+U` 打开源代码
2. `Ctrl+F` 搜索 `js_content`
3. 从 `id="js_content"` 开始选中 HTML → `Ctrl+C` 复制 → 粘贴给 Claude Code

### 添加新排版主题

1. 使用 `/extract-style` 从喜欢的推文中提取样式

3. 下次使用 `/paper2wechat` 时即可选择新类型

## 十、脚本参考

### extract_figures.py

PDF 图片提取脚本，基于 PyMuPDF。

```bash
python .claude/scripts/extract_figures.py <pdf_path> <output_dir>
```

- 提取所有大于 200x200px 的图片
- 自动匹配 Figure 编号命名（fig1.png, fig2.jpeg...）
- 输出 `figures_meta.json` 元信息文件

### md2wechat.py

Markdown → 微信公众号 HTML 转换器。

```bash
# 使用 doocs 主题
python .claude/scripts/md2wechat.py input.md --theme default --primary-color "#20B2AA"

# 使用 doocs 优雅主题
python .claude/scripts/md2wechat.py input.md --theme grace

# 使用自定义模板
python .claude/scripts/md2wechat.py input.md --template .claude/templates/themes/custom_yijiyin_20250510.css

# 指定输出路径
python .claude/scripts/md2wechat.py input.md -o output/文章标题.html
```

**参数说明**：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `input` | 输入 Markdown 文件路径 | 必填 |
| `--output, -o` | 输出 HTML 文件路径 | 与输入同名.html |
| `--theme, -t` | doocs 主题：`default` / `grace` / `simple` | `default` |
| `--template` | 自定义 CSS 模板文件路径 | 无 |
| `--primary-color, -c` | 主题色 | `#20B2AA` |

## 十一、常见问题

| 问题 | 解决方案 |
|------|----------|
| PDF 是扫描版（提取文本为空） | 先用 OCR 工具处理（Adobe Acrobat、ABBYY FineReader） |
| 图片提取不到 | 手动从 PDF 导出图片到 `output/figures/` 目录 |
| 微信推文抓取失败 | 部署 wechat-parser-mcp 或使用书签方案（见第八章） |
| MCP 导入错误 `attempted relative import` | 在 wechat-parser-mcp 目录下执行 `pip install -e .` |
| Playwright 未安装 | `pip install playwright && playwright install chromium` |
| MCP 抓取内容被截断（仅2000字符） | `detail` 参数默认值为 `concise`，会截断正文。使用 `"detail": "detailed"` 获取完整内容。详见下方"常见故障排查" |
| Windows 终端 MCP 输出中文乱码 | 使用 `python -X utf8` 或将输出写入文件（`encoding='utf-8'`），避免终端 GBK 编码转换 |
| MCP 工具在 Claude Code 中不可见 | 检查 mcp.json 路径是否正确。项目级配置在 `.claude/mcp.json`，用户级在 `~/.claude/mcp.json`。配置后需重启 Claude Code |
| 直接调用 Python 模块报 `coroutine was never awaited` | `parse_article` 是 async 函数，需用 `asyncio.run(main())` 包装调用 |
